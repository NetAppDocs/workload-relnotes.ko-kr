=== SSD 용량 증가 관리 모드 및 자동화

Workload Factory는 이제 운영 환경 설정에 맞는 SSD 용량 증가 관리 모드를 제공합니다.

* *자동화*
* *권장 사항*
* *없음*


현재 모드와 해당 매개변수는 파일 시스템 세부 정보 페이지의 SSD 사용량 그래프 상단에 표시됩니다. 운영상의 필요에 따라 언제든지 모드를 전환할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#change-ssd-management-mode["SSD 관리 모드에 대해 알아보십시오"^].



[[bulk-edit-ssd-capacity-increase-management-parameters]]
=== SSD 용량 증가 관리 파라미터 일괄 편집

이제 Overview 대시보드에서 여러 파일 시스템에 걸쳐 SSD 용량 증가 관리 파라미터를 조정할 수 있습니다. Clusters 테이블에서 하나 이상의 클러스터를 선택하고 Threshold(10~90%), Lookback(1~200시간), Ahead(1~200시간) 값을 일괄적으로 편집하세요.

편집 대화 상자는 사용자의 선택에 따라 자동으로 채워집니다. 선택한 모든 파일 시스템이 동일한 매개변수 값을 공유하는 경우 해당 값이 표시되고, 그렇지 않으면 기본값이 사용됩니다. 이를 통해 환경 전체에 일관된 용량 설정을 적용할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#bulk-edit-ssd-management-parameters["매개 변수를 일괄 편집하는 방법을 알아보십시오"^].



[[capacity-decision-points-visualization]]
=== 용량 결정 지점 시각화

이제 SSD 사용량 그래프에 용량 권장 사항이 생성되었거나 자동화된 작업이 수행된 시점을 나타내는 의사 결정 지점이 표시됩니다.

* *권장 결정 지점*: 마우스를 올려놓으면 타임스탬프, 권장 증가 크기 및 현재 용량을 볼 수 있습니다.
* *자동화 결정 지점*: 마우스를 올려놓으면 타임스탬프, 성공 또는 실패 상태, 크기 증가, 새 애그리게이트 크기를 볼 수 있습니다. 실패한 시도에는 실패 이유가 포함됩니다.


의사 결정 시점은 30분마다 발생할 수 있으며, 선택한 시간 범위에 따라 개별적으로 또는 통합되어 표시됩니다.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#analyze-capacity-trends-and-decision-points["용량 결정 시점에 대해 알아보십시오"^].



[[link-configured-status-in-clusters-table]]
=== 클러스터 테이블의 링크 구성 상태

개요 대시보드의 클러스터 테이블에 이제 각 파일 시스템에 대해 FSx for ONTAP 링크가 구성되었는지 여부를 보여주는 연결된 링크 열이 추가되었습니다. 연결 상태는 매일 검증되므로 링크 구성이 필요하거나 연결 문제가 있을 수 있는 파일 시스템을 쉽게 파악할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html["개요 대시보드 보기"^].



[[support-for-aws-govcloud-us-in-netapp-workload-factory]]
=== NetApp Workload Factory에서 AWS GovCloud (US) 지원

NetApp Workload Factory는 AWS GovCloud(미국) 계정을 지원합니다. GovCloud 계정을 사용하면 AWS GovCloud(미국) 리전에 스토리지 워크로드를 배포하고 관리할 수 있습니다.
