= NetApp Workload Factory의 최근 변경 사항
:allow-uri-read:

[role="lead"]
NetApp Workload Factory에 포함된 워크로드의 최신 변경 사항에 대해 알아보세요.



== 2026년 5월 3일



=== NetApp ONTAP용 Amazon FSx



==== 블록 디바이스 클론

Workload Factory는 블록 디바이스용 클론을 제공하여 용량, 액세스 및 수명 주기를 제어하면서 쓰기 가능하고 효율적인 데이터 복사본을 신속하게 프로비저닝할 수 있도록 지원합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["NetApp Workload Factory에서 블록 스토리지를 관리합니다"]



==== 블록 디바이스의 EC2 태그

블록 장치 테이블에 이제 각 블록 장치를 사용하는 EC2 호스트의 태그를 보여주는 열이 추가되었습니다. 이를 통해 문제 해결이나 감사 중에 어떤 호스트가 각 블록 장치를 사용하는지 쉽게 파악할 수 있습니다.

각 블록 디바이스에 태그를 생성하려면 제공된 사용자 스크립트를 사용하여 EC2 인스턴스를 배포하십시오.

link:https://github.com/NetApp/FSx-ONTAP-samples-scripts/tree/main/Management-Utilities/ec2-user-data-iscsi-create-and-mount["블록 디바이스에 태그를 할당하는 EC2 사용자 데이터 스크립트"]



==== FSx for ONTAP 파일 시스템의 용량 추세

FSx for ONTAP 파일 시스템의 개요에서 SSD 용량 추세를 직접 확인하세요. 용량 관리를 위한 새로운 카드는 지난달 사용량 추세에 대한 포괄적인 정보를 제공합니다. 이 분석에는 실제 사용 용량과 할당된 용량 간의 비교가 포함되어 리소스 활용률을 명확하게 평가할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-capacity-automatically.html["NetApp Workload Factory에서 용량을 자동으로 관리하세요"]



=== 설정 및 관리



==== Workload Factory의 이메일 알림

이제 Amazon SNS 토픽을 구성하지 않고도 이메일을 통해 Workload Factory 알림을 받을 수 있습니다. 계정에 연결된 모든 이메일 주소로 알림을 보내려면 이메일 알림을 활성화하세요.

Workload Factory 콘솔 또는 NetApp Console에서 이메일 알림을 구성할 수 있습니다. NetApp Console에서는 종 모양 아이콘 아래에 표시되는 NetApp Console 알림을 구성하는 옵션도 제공합니다.

NetApp Console에서 알림을 구성하면 NetApp Console이 알림 전달을 담당하게 됩니다. 중복 알림을 방지하기 위해 Workload Factory 콘솔의 알림 설정은 읽기 전용으로 변경됩니다.



==== EDA 지연 시간에 대한 새 알림

NetApp Workload Factory 알림 서비스에는 EDA 작업의 지연 시간이 특정 임계값을 초과할 때 알려주는 EDA 지연 시간 알림 기능이 포함되어 있습니다.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Workload Factory 알림 구성"]



=== EDA 워크로드



==== 지연 이벤트에 대한 이메일 및 SNS 알림

이제 FSx for ONTAP 볼륨에서 지연 시간 이벤트가 감지되면 자동 알림을 받을 수 있습니다. Workload Factory 설정에서 이메일 또는 Amazon SNS 알림을 구성하여 볼륨이 구성된 지연 시간 임계값을 위반할 경우 실시간 알림을 받을 수 있습니다.

알림은 파일 시스템 단위로 전송되며 영향을 받는 모든 볼륨에 대한 세부 정보, 심각도 수준, 임계값 비교 및 조사를 위한 지연 시간 페이지로의 직접 링크를 포함합니다.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["지연 시간 알림을 구성하는 방법을 알아보세요."^].



==== 대화형 그래프를 사용한 지연 시간 추세 시각화

지연 시간 분석 패널에 이제 포괄적인 지연 시간 분석을 위한 탭 인터페이스가 포함되었습니다.

* *개요*: 지연 시간을 유발하는 구성 요소를 보여주는 자동화된 기본 분석 결과를 확인합니다.
* *시간 경과*: 여러 시간대(1시간, 3시간, 12시간, 24시간 또는 72시간)에 걸쳐 대화형 그래프로 볼륨 지연 시간 동작을 시각화합니다


*시간 경과에 따른 변화* 탭에는 원시 CloudWatch 지연 시간 데이터(알람 발생 원인에 따라 읽기 또는 쓰기)가 경고 및 위험 임계값 선, 위반 표시기, 각 위반에 대한 세부 메트릭과 함께 표시됩니다.

이 포괄적인 시각화를 통해 패턴을 식별하고, 지연 시간 문제가 반복적인지 또는 일회성인지 파악하며, 해결 방법에 대해 더 많은 정보를 바탕으로 의사 결정을 내릴 수 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["지연 시간 추세를 분석하는 방법을 알아보십시오"^].



== 2026년 4월 6일



=== NetApp ONTAP용 Amazon FSx



==== 잘 설계된 분석 업데이트

Workload Factory는 FSx for ONTAP 파일 시스템에서 *비활성 NAS 볼륨*을 분석합니다. 구성 분석을 통해 활발하게 사용되지 않는 NAS 볼륨을 식별하고 스토리지 활용도를 최적화하고 비용을 절감하기 위한 권장 사항을 제시합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["잘 설계된 FSx for ONTAP 파일 시스템 구현"]



==== S3 액세스 포인트용 저널 테이블 기능

저널 테이블 기능은 FSx ONTAP S3 객체 스토리지를 대규모로 관리할 수 있는 방법을 제공하여 버킷의 객체에 대한 사용자 액세스 이벤트 및 작업을 쉽게 감사할 수 있도록 합니다. Workload Factory를 사용하면 버킷과 S3 테이블에 대한 자동 스캔을 설정하여 모든 객체, 객체 세부 정보 및 사용자 작업을 표시하는 이벤트 기록을 유지할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["NetApp Workload Factory용 저널 테이블 인프라를 설정합니다"]



==== 기존 igroup 관리 지원

Workload Factory는 기존 igroup 관리를 지원합니다. 블록 디바이스와 호스트 이니시에이터를 추가하거나 제거하여 배포된 워크로드를 정시에 제공할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["NetApp Workload Factory에서 블록 스토리지를 관리합니다"]



==== FlexVol 볼륨에 대한 애그리게이트 지정 지원

Workload Factory는 선택한 애그리게이트에 FlexVol 볼륨을 생성하는 기능을 지원합니다. 이를 통해 모든 파일 시스템 리소스에 용량 및 성능 로드를 분산할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["NetApp Workload Factory에서 FlexVol 볼륨 생성"]



==== 블록 디바이스의 마지막 액세스 시간

Workload Factory는 블록 장치의 마지막 액세스 시간을 보고합니다. 이 시간은 Workload Factory에서 igroup에 마지막으로 액세스한 시간을 기준으로 합니다. 각 블록 장치의 마지막 사용 시간을 알면 리소스를 최적화할 수 있습니다.



=== 데이터베이스 워크로드



==== Oracle을 위한 잘 설계된 분석

잘 설계된 아키텍처 분석에는 Oracle에 대한 다음과 같은 새로운 구성이 포함됩니다.

* *Oracle Critical Patch Updates*: 지원되는 Oracle 자체 관리형 데이터베이스 설치에 누락된 보안 패치를 평가합니다.
* *지역 간 복제*: Oracle 데이터베이스를 제공하는 FSx for ONTAP 파일 시스템에 대해 지역 간 복제(CRR)가 활성화되어 있는지 확인합니다.
* *애플리케이션 정합성 보장 스냅샷*: Oracle 데이터베이스가 NetApp SnapCenter를 통해 애플리케이션 정합성 보장 스냅샷으로 보호되는지 확인합니다.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다."]



==== Oracle에 대한 일회성 평가

Workload Factory는 이제 필요할 때마다 Oracle 데이터베이스 배포 구성을 분석할 수 있는 일회성 평가 기능을 제공합니다(신규 배포, 마이그레이션 후, 문제 해결 등). 배포 구성을 최적화하고 모범 사례를 준수하는 데 도움이 되는 권장 사항을 받아볼 수 있습니다. 이 기능을 통해 설정 프로세스를 완전히 수행하지 않고도 잘 설계된 기능을 사용해 볼 수 있습니다. Workload Factory에 자격 증명을 저장하거나, 데이터베이스 리소스를 등록하거나, 외부 연결을 설정하지 않고도 데이터베이스를 평가할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Oracle 데이터베이스 배포에 대한 일회성 평가를 수행하십시오"]



==== 여러 Oracle 데이터베이스의 비용 절감 효과 개선

Oracle의 비용 절감 분석을 개선하여 여러 Oracle 온프레미스 데이터베이스를 하나의 FSx for ONTAP 파일 시스템에 통합된 스토리지 솔루션으로 제공합니다. 이러한 통합은 여러 데이터베이스에 필요한 파일 시스템 수를 줄여 스토리지 비용을 최적화하고 비용 절감 효과를 향상시킵니다.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Oracle 온프레미스에서 감지된 데이터베이스에 대한 비용 절감 효과를 살펴보십시오"]

=== EDA 워크로드



==== EDA 용량 권장 사항

Workload Factory는 이제 EDA 인벤토리에 있는 각 FSx for ONTAP 클러스터의 SSD 용량을 자동으로 분석하고 조정이 필요할 때 용량 권장 사항을 생성합니다. 알고리즘은 각 클러스터에 대해 24시간마다 한 번씩 실행됩니다.

권장 사항이 확인되면:

* 즉시 이메일 또는 WAD 알림을 받게 됩니다(주간 요약에는 포함되지 않음).
* Clusters 테이블에서 영향을 받는 클러스터에 전구 표시등이 나타납니다.
* Overview 대시보드의 요약에는 활성 권장 사항이 있는 클러스터의 총 수가 표시됩니다.


활성 권장 사항이 있는 클러스터를 선택하여 권장 사항을 확인하세요. 권장 사항에는 제안된 변경 사항과 그 이유가 설명되어 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["용량 권장 사항에 대해 자세히 알아보십시오"^].



==== 자동화된 AI 지연 시간 분석

지연 시간 분석에 이제 AI 에이전트 분석이 포함됩니다.

데이터 또는 클러스터 지연 시간과 관련된 복잡한 시나리오의 경우 AI 에이전트 분석을 호출하여 다음을 수신할 수 있습니다.

* 상세한 근본 원인 식별
* 영향을 받는 EC2 클라이언트 목록
* 구체적인 개선 조치


AI 에이전트 분석에는 Amazon Bedrock 모델 ARN 구성이 필요하며 기본 분석에서 심층 조사를 권장하는 시나리오에 대해 온디맨드로 제공됩니다. AI 에이전트는 과부하 볼륨, 최적화되지 않은 마운트 포인트 구성, FlexGroup 리밸런싱 필요성 및 스케일아웃 요구 사항과 같은 문제를 식별할 수 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["AI 에이전트 분석에 대해 자세히 알아보십시오"^].

=== VMware 워크로드



==== Google Cloud VMware Engine용 마이그레이션 어드바이저 지원 소개

NetApp Workload Factory가 이제 Google Cloud VMware Engine을 지원합니다. Google Cloud NetApp Volumes를 외부 NFS 데이터스토어로 사용하여 온프레미스 VMware 워크로드를 Google Cloud VMware Engine으로 마이그레이션할 수 있습니다. 이를 통해 애플리케이션을 재구성하거나 플랫폼을 변경할 필요 없이 비용을 최적화하고 VMware 환경을 더욱 효과적으로 제어할 수 있습니다.

Google Cloud VMware Engine 마이그레이션 어드바이저는 다음을 제공합니다.

* VE1 및 VE2 노드 유형을 지원하는 자동화된 Google Cloud VMware Engine 클러스터 크기 조정 및 설계
* vSAN 내부 데이터스토어와 Google Cloud NetApp Volumes 외부 NFS 데이터스토어 간의 VM 레이아웃 최적화.
* vSAN 단독 구성과 Google Cloud NetApp Volumes 강화 구성 간의 상세한 비용 비교 및 총소유비용(TCO) 분석.
* Standard, Premium 및 Extreme Google Cloud NetApp Volumes 서비스 레벨을 지원합니다.
* Terraform을 사용한 자동화된 프로비저닝을 위한 Infrastructure-as-Code 배포.

=== 데이터베이스 워크로드



==== SQL Server에 대한 일회성 평가

Workload Factory는 이제 필요할 때마다 SQL Server 데이터베이스 배포 구성을 분석하는 일회성 평가를 제공합니다. 신규 배포, 마이그레이션 후, 문제 해결 등 다양한 상황에서 활용할 수 있습니다. 배포 구성을 최적화하고 모범 사례를 준수하는 데 도움이 되는 권장 사항을 받아볼 수 있습니다. 이 기능을 통해 설정 프로세스를 완전히 수행하지 않고도 잘 설계된 기능을 사용해 볼 수 있습니다. Workload Factory에 자격 증명을 저장하거나, 데이터베이스 리소스를 등록하거나, 외부 연결을 설정하지 않고도 데이터베이스를 평가할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["데이터베이스 배포에 대한 일회성 평가를 수행합니다"]



==== Microsoft SQL Server의 Always On 가용성 그룹 및 Oracle 데이터베이스의 Data Guard에 대한 등록 지원

이제 NetApp Workload Factory의 데이터베이스 워크로드에서 Always On 가용성 그룹(AOAG) 배포 모델을 사용하는 Microsoft SQL Server 인스턴스와 Data Guard 배포 모델을 사용하는 Oracle 데이터베이스를 검색하고 등록할 수 있습니다. 이 개선 사항을 통해 Workload Factory 콘솔에서 SQL Server 및 Oracle 리소스를 더욱 효율적으로 관리할 수 있습니다.

AOAG 인스턴스와 Data Guard 데이터베이스를 등록한 후에는 리소스 세부 정보 보기 및 오류 로그 분석과 같은 NetApp Workload Factory for Databases의 다양한 기능을 활용할 수 있습니다.

Workload Factory for Databases는 AOAG 배포 모델을 사용하는 Microsoft SQL Server 인스턴스에 대한 well-architected 분석도 지원합니다. 가용성 그룹에 속한 SQL Server 인스턴스의 구성을 분석하고 배포 구성을 최적화하고 모범 사례를 준수하도록 권장 사항을 받을 수 있습니다. 이 기능을 사용하려면 AOAG 인스턴스를 등록하십시오.

link:https://docs.netapp.com/us-en/workload-databases/register-instance.html#register-a-sql-server-instance["데이터베이스용 Workload Factory에 리소스를 등록합니다"]
