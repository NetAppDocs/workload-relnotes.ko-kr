= NetApp Workload Factory의 최근 변경 사항
:allow-uri-read:

:imagesdir: ./media/


[role="lead"]
NetApp Workload Factory에 포함된 워크로드의 최신 변경 사항에 대해 알아보세요.



== 2026년 4월 6일



=== NetApp ONTAP용 Amazon FSx



==== 잘 설계된 분석 업데이트

Workload Factory는 FSx for ONTAP 파일 시스템에서 *비활성 NAS 볼륨*을 분석합니다. 구성 분석을 통해 활발하게 사용되지 않는 NAS 볼륨을 식별하고 스토리지 활용도를 최적화하고 비용을 절감하기 위한 권장 사항을 제시합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["잘 설계된 FSx for ONTAP 파일 시스템 구현"]



==== S3 액세스 포인트용 저널 테이블 기능

저널 테이블 기능은 FSx ONTAP S3 객체 스토리지를 대규모로 관리할 수 있는 방법을 제공하여 버킷의 객체에 대한 사용자 액세스 이벤트 및 작업을 쉽게 감사할 수 있도록 합니다. Workload Factory를 사용하면 버킷과 S3 테이블에 대한 자동 스캔을 설정하여 모든 객체, 객체 세부 정보 및 사용자 작업을 표시하는 이벤트 기록을 유지할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["NetApp Workload Factory용 저널 테이블 인프라를 설정합니다"]



==== 기존 igroup 관리 지원

Workload Factory는 기존 igroup 관리를 지원합니다. 블록 디바이스와 호스트 이니시에이터를 추가하거나 제거하여 배포된 워크로드를 정시에 제공할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["NetApp Workload Factory에서 블록 스토리지를 관리합니다"]



==== FlexVol 볼륨에 대한 애그리게이트 지정 지원

Workload Factory는 선택한 애그리게이트에 FlexVol 볼륨을 생성하는 기능을 지원합니다. 이를 통해 모든 파일 시스템 리소스에 용량 및 성능 로드를 분산할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["NetApp Workload Factory에서 FlexVol 볼륨 생성"]



=== 데이터베이스 워크로드



==== Oracle을 위한 잘 설계된 분석

잘 설계된 아키텍처 분석에는 Oracle에 대한 다음과 같은 새로운 구성이 포함됩니다.

* *Oracle Critical Patch Updates*: 지원되는 Oracle 자체 관리형 데이터베이스 설치에 누락된 보안 패치를 평가합니다.
* *지역 간 복제*: Oracle 데이터베이스를 제공하는 FSx for ONTAP 파일 시스템에 대해 지역 간 복제(CRR)가 활성화되어 있는지 확인합니다.
* *애플리케이션 정합성 보장 스냅샷*: Oracle 데이터베이스가 NetApp SnapCenter를 통해 애플리케이션 정합성 보장 스냅샷으로 보호되는지 확인합니다.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다."]



==== Oracle에 대한 일회성 평가

Workload Factory는 이제 필요할 때마다 Oracle 데이터베이스 배포 구성을 분석할 수 있는 일회성 평가 기능을 제공합니다(신규 배포, 마이그레이션 후, 문제 해결 등). 배포 구성을 최적화하고 모범 사례를 준수하는 데 도움이 되는 권장 사항을 받아볼 수 있습니다. 이 기능을 통해 설정 프로세스를 완전히 수행하지 않고도 잘 설계된 기능을 사용해 볼 수 있습니다. Workload Factory에 자격 증명을 저장하거나, 데이터베이스 리소스를 등록하거나, 외부 연결을 설정하지 않고도 데이터베이스를 평가할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Oracle 데이터베이스 배포에 대한 일회성 평가를 수행하십시오"]



==== 여러 Oracle 데이터베이스의 비용 절감 효과 개선

Oracle의 비용 절감 분석을 개선하여 여러 Oracle 온프레미스 데이터베이스를 하나의 FSx for ONTAP 파일 시스템에 통합된 스토리지 솔루션으로 제공합니다. 이러한 통합은 여러 데이터베이스에 필요한 파일 시스템 수를 줄여 스토리지 비용을 최적화하고 비용 절감 효과를 향상시킵니다.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Oracle 온프레미스에서 감지된 데이터베이스에 대한 비용 절감 효과를 살펴보십시오"]

== 2026년 3월 9일



=== VMware 워크로드



==== Google Cloud VMware Engine용 마이그레이션 어드바이저 지원 소개

NetApp Workload Factory가 이제 Google Cloud VMware Engine을 지원합니다. Google Cloud NetApp Volumes를 외부 NFS 데이터스토어로 사용하여 온프레미스 VMware 워크로드를 Google Cloud VMware Engine으로 마이그레이션할 수 있습니다. 이를 통해 애플리케이션을 재구성하거나 플랫폼을 변경할 필요 없이 비용을 최적화하고 VMware 환경을 더욱 효과적으로 제어할 수 있습니다.

Google Cloud VMware Engine 마이그레이션 어드바이저는 다음을 제공합니다.

* VE1 및 VE2 노드 유형을 지원하는 자동화된 Google Cloud VMware Engine 클러스터 크기 조정 및 설계
* vSAN 내부 데이터스토어와 Google Cloud NetApp Volumes 외부 NFS 데이터스토어 간의 VM 레이아웃 최적화.
* vSAN 단독 구성과 Google Cloud NetApp Volumes 강화 구성 간의 상세한 비용 비교 및 총소유비용(TCO) 분석.
* Standard, Premium 및 Extreme Google Cloud NetApp Volumes 서비스 레벨을 지원합니다.
* Terraform을 사용한 자동화된 프로비저닝을 위한 Infrastructure-as-Code 배포.




== 2026년 3월 2일



=== 데이터베이스 워크로드



==== SQL Server에 대한 일회성 평가

Workload Factory는 이제 필요할 때마다 SQL Server 데이터베이스 배포 구성을 분석하는 일회성 평가를 제공합니다. 신규 배포, 마이그레이션 후, 문제 해결 등 다양한 상황에서 활용할 수 있습니다. 배포 구성을 최적화하고 모범 사례를 준수하는 데 도움이 되는 권장 사항을 받아볼 수 있습니다. 이 기능을 통해 설정 프로세스를 완전히 수행하지 않고도 잘 설계된 기능을 사용해 볼 수 있습니다. Workload Factory에 자격 증명을 저장하거나, 데이터베이스 리소스를 등록하거나, 외부 연결을 설정하지 않고도 데이터베이스를 평가할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["데이터베이스 배포에 대한 일회성 평가를 수행합니다"]



==== Microsoft SQL Server의 Always On 가용성 그룹 및 Oracle 데이터베이스의 Data Guard에 대한 등록 지원

이제 NetApp Workload Factory의 데이터베이스 워크로드에서 Always On 가용성 그룹(AOAG) 배포 모델을 사용하는 Microsoft SQL Server 인스턴스와 Data Guard 배포 모델을 사용하는 Oracle 데이터베이스를 검색하고 등록할 수 있습니다. 이 개선 사항을 통해 Workload Factory 콘솔에서 SQL Server 및 Oracle 리소스를 더욱 효율적으로 관리할 수 있습니다.

AOAG 인스턴스와 Data Guard 데이터베이스를 등록한 후에는 리소스 세부 정보 보기 및 오류 로그 분석과 같은 NetApp Workload Factory for Databases의 다양한 기능을 활용할 수 있습니다.

Workload Factory for Databases는 AOAG 배포 모델을 사용하는 Microsoft SQL Server 인스턴스에 대한 well-architected 분석도 지원합니다. 가용성 그룹에 속한 SQL Server 인스턴스의 구성을 분석하고 배포 구성을 최적화하고 모범 사례를 준수하도록 권장 사항을 받을 수 있습니다. 이 기능을 사용하려면 AOAG 인스턴스를 등록하십시오.

link:https://docs.netapp.com/us-en/workload-databases/register-instance.html#register-a-sql-server-instance["데이터베이스용 Workload Factory에 리소스를 등록합니다"]



==== Oracle을 위한 잘 설계된 분석

체계적인 분석에는 운영 체제에 최신 패치가 있는지 확인하고 설치 방법을 알려주는 Oracle용 *운영 체제 패치*가 포함되어 있습니다.

또한, Oracle용 Well-Architected 분석 기능은 스토리지 레이아웃과 같은 구성이 다른 구성에 종속되어 있을 때 알려줍니다. 이 알림은 연결된 구성을 최적화하여 제공되는 모든 권장 사항이 최적화되도록 하는 데 필요한 조치를 안내합니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다."]



==== Oracle 온프레미스 데이터베이스 환경 비용 분석 및 마이그레이션 계획

Workload Factory for Databases는 이제 Oracle 온프레미스 데이터베이스 마이그레이션을 Amazon FSx for NetApp ONTAP로 분석하고 계획하는 데 도움을 줍니다. 비용 절감 계산기를 사용하여 클라우드에서 Oracle 온프레미스 데이터베이스 환경을 운영하는 데 드는 비용을 추산하고 온프레미스 데이터베이스 환경을 클라우드로 마이그레이션하기 위한 권장 사항을 검토할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html["온프레미스 데이터베이스 환경의 비용 절감 효과를 살펴보세요."]



==== Workload Factory에서 사용할 수 있는 새로운 Ask Me 북마크

Workload Factory 콘솔의 모든 화면에 새로운 Ask Me 북마크를 추가했습니다. 이 개선 사항을 통해 필요할 때 언제든지 Ask Me에 더 쉽고 빠르게 액세스할 수 있습니다. Ask Me는 사용자의 워크로드 환경에 대한 질문을 하고, 환경에서 직접 맞춤형 인사이트를 얻고, 이전 대화를 참조할 수 있도록 지원하는 AI 비서입니다.

새로운 북마크를 클릭하면 어느 페이지에서든 Ask Me를 열 수 있습니다. 현재 작업 중인 내용을 방해하지 않고 사이드 패널에 Ask Me가 실행되어, 현재 작업과 관련된 간단한 설명과 추천 사항을 제공합니다.

link:https://docs.netapp.com/us-en/workload-setup-admin/workload-factory-overview.html#features["Ask Me에 대해 자세히 알아보기"^]



== 2026년 3월 1일



=== NetApp ONTAP용 Amazon FSx



==== 잘 설계된 분석 업데이트

Workload Factory는 다음 구성에 대해 FSx for ONTAP 파일 시스템을 분석합니다.

* SSD 용량 활용률 저하: SSD 용량 계층의 활용률이 낮은지 확인하고 비용 최적화를 위해 SSD 용량을 줄이는 것을 권장합니다.
* 볼륨 백업 예약: Workload Factory는 구성 권장 사항을 개선하고 FSx for ONTAP 백업 또는 AWS Backup을 활성화하는 옵션을 추가했습니다. FSx for ONTAP 백업은 파일 시스템의 모든 볼륨에 기본 정책을 적용하여 최대 90일 동안 보관되는 일일 백업을 생성합니다. AWS Backup을 사용하면 백업 계획에서 각 볼륨에 대해 서로 다른 보존 기간을 선택할 수 있습니다.




==== Workload Factory는 복제를 위한 AI 및 ML 사용 사례를 지원합니다

Workload Factory는 온프레미스 ONTAP 데이터를 FSx for ONTAP로 복제하고 AWS 인공지능(AI) 및 머신러닝(ML) 서비스를 위한 S3 액세스 포인트를 생성하여 AI/ML 워크플로를 가속화하는 기능을 지원합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["AI 및 ML 사용 사례를 위해 온프레미스 데이터를 복제합니다"]



==== 기존 S3 액세스 포인트 편집

볼륨에 연결된 기존 S3 액세스 포인트의 사용자 및 사용자 유형을 변경할 수 있습니다. 또한 액세스 포인트의 메타데이터를 활성화 또는 비활성화할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/edit-s3-access-points.html["NetApp Workload Factory에서 S3 액세스 포인트 편집"]



==== 블록 스토리지 다이어그램 추가

블록 스토리지를 사용하는 경우 블록 스토리지 다이어그램을 통해 블록 장치가 노드 및 igroup에 어떻게 연결되는지 시각적으로 확인할 수 있습니다. 이 다이어그램은 블록 스토리지를 이해하고 관리하는 데 도움이 됩니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["NetApp Workload Factory에서 블록 스토리지를 관리합니다"]



==== 자동 용량 관리 기능으로 업데이트

이번 업데이트에서는 해당 기능을 자동 용량 관리와 용량 알림이라는 두 가지 별개의 기능으로 분리했습니다.

자동 용량 관리는 최적의 성능을 유지하기 위해 파일 시스템 용량을 자동으로 추가하거나 제거합니다. 새로운 옵션에는 증분 모드와 적응형 모드가 포함됩니다. 증분 모드에서는 시스템이 임계값에 도달할 때마다 고정된 양만큼 용량을 늘리고, 적응형 모드에서는 기간별 데이터를 사용하여 용량을 예측하고 조정합니다.

용량 알림은 용량 조정이 발생하거나 파일 시스템이 설정된 한도에 도달할 때마다 알려주므로 문제가 발생하기 전에 스토리지를 관리할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/enable-auto-capacity-management.html["NetApp Workload Factory의 자동 용량 관리 및 용량 알림에 대해 알아보십시오"]

Workload Factory는 다음 구성에 대해 FSx for ONTAP 파일 시스템을 분석합니다.

* 캐시 볼륨 크기 최적화: 캐시 볼륨에서 볼륨 autosize 및 scrubbing이 활성화되어 있는지 확인하여 최적의 크기를 유지하고 캐시가 핫 데이터에 집중하여 최고 효율성을 달성합니다.

* 스토리지 VM 논리적 보고: 스토리지 VM의 기본 보고 설정이 논리적으로 설정되어 있는지 확인하여 볼륨 수준에서 스토리지 사용량을 더 잘 파악할 수 있도록 합니다.

==== Amazon Elastic VMware Service에 대한 Well-Architected 분석

Workload Factory는 이제 Amazon Elastic VMware Service(EVS) 환경에 대한 자동화된 well-architected 분석 기능을 제공합니다. 매일 실행되는 검사를 통해 구성 불일치를 식별하고 최적의 운영, 보안 및 비용 효율성을 유지하는 데 도움이 되는 자세한 해결 권장 사항을 제공합니다.

AWS API를 사용하여 스캔이 수행되므로 vSphere 자격 증명이나 vCenter 연결이 필요하지 않습니다. 결과는 EVS 환경 세부 정보 내의 새로운 *Well-architected status* 탭에서 확인할 수 있습니다.

이번 릴리스에는 다음에 대한 인사이트가 포함됩니다.

* *인스턴스 중지/종료 보호 상태*: EC2 중지 또는 종료 보호가 적용되지 않은 EVS 노드를 식별합니다. EC2 콘솔에서 EVS 노드를 중지하거나 종료하면 가상 머신 데이터에 접근할 수 없거나 데이터가 손실될 수 있습니다.
* *클러스터 노드 파티션 배치 정렬*: AWS 가용 영역 내 파티션에 장애가 발생할 경우 상당한 처리 능력 손실이나 다운타임을 초래할 수 있는 파티션 정렬 오류를 감지합니다. 각 인사이트에는 심각도 수준, 영향을 받는 리소스 정보를 포함한 자세한 결과, AWS 모범 사례에 기반한 단계별 해결 절차가 포함됩니다.


link:view-evs-well-architected.html["잘 설계된 구성 구현"]

==== Well-Architected 업데이트

NetApp Workload Factory에는 Elastic VMware Service(EVS) 워크로드에 대한 아키텍처 평가 기능이 포함되어 있으며 스토리지 및 데이터베이스 워크로드에 대한 새로운 구성이 추가되었습니다.

* *VMware 워크로드* NetApp Workload Factory는 잘 설계된 Amazon Elastic VMware Service(EVS) 워크로드 운영을 위한 모범 사례 및 권장 사항을 제공합니다.
+
link:https://docs.netapp.com/us-en/workload-vmware/view-evs-well-architected.html["잘 설계된 EVS 구성 구현"^]

* *스토리지 워크로드* 스토리지 성능 및 비용에 대한 더 많은 통찰력을 얻을 수 있도록 스토리지 워크로드의 잘 설계된 기능에 여러 가지 새로운 구성이 추가되었습니다.
+
** 스토리지 VM 논리적 보고
** 캐시 볼륨 크기 최적화
** 고아 블록 디바이스
+
link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["스토리지 워크로드에 적합한 아키텍처의 파일 시스템 구성 구현"^]



* *데이터베이스 워크로드* Workload Factory for Databases에는 Direct NFS(dNFS)를 활성화 및 설정하여 I/O 성능을 향상시키고 호스트 및 스토리지 시스템의 부하를 줄이기 위한 Oracle용 새로운 스토리지 구성이 포함되어 있습니다.

=== EDA 워크로드



==== 사용자 지정 태그를 사용한 향상된 대시보드 필터링

이제 AWS 태그를 기반으로 EDA 대시보드에서 최대 5개의 사용자 지정 필터를 구성할 수 있습니다. 각 사용자 지정 필터에는 레이블 이름, AWS 태그 키 및 선택 유형(단일 또는 다중 선택)이 포함됩니다.

다중 선택 기능을 사용하면 여러 값을 동시에 선택할 수 있지만, 단일 선택 기능은 한 번에 하나의 값만 선택할 수 있도록 제한합니다. 사용자 지정 필터는 구성한 순서대로 표시되므로 자주 사용하는 필터를 쉽게 정리할 수 있습니다.

사용자 지정 필터를 구성하지 않으면 기본 필터(파일 시스템, 볼륨 유형 및 시간 범위)를 계속 사용할 수 있으므로 대시보드를 계속 보고 상호 작용할 수 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-configure-dashboard.html["사용자 지정 필터 구성에 대해 자세히 알아보십시오"^].



==== 세부적인 성능 분석을 위한 볼륨 세부 정보 보기

이제 대시보드는 전체 보기와 볼륨 보기의 두 가지 보기 모드를 제공합니다. 전체 보기에서는 모든 볼륨에 대한 집계 지표를 표시하고, 볼륨 보기에서는 시간 경과에 따른 상위 10개 볼륨의 개별 성능을 보여줍니다.

볼륨 보기에서 마우스 오버 시 나타나는 대화형 툴팁은 볼륨 이름, 측정항목, 시간별 값 등 각 볼륨에 대한 자세한 정보를 제공합니다. 동일한 볼륨이 여러 구성요소에 걸쳐 표시되는 경우, 일관된 색상 코드를 통해 다양한 측정항목에서 특정 볼륨을 쉽게 추적할 수 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-configure-dashboard.html#view-volume-details["볼륨 세부 정보 보기에 대해 자세히 알아보십시오"^].



==== 사전 예방적 성능 모니터링을 위한 지연 시간 분석

지연 시간 분석 기능을 사용하면 FSx for ONTAP 파일 시스템 전체에서 볼륨 읽기 및 쓰기 지연 시간을 모니터링할 수 있습니다. 사용자 지정 가능한 경고 및 중요 이벤트 임계값을 구성하여 EDA 워크로드에 영향을 미치기 전에 성능 병목 현상을 사전에 파악할 수 있습니다.

지연 시간 이벤트 테이블에는 모든 경고 및 위험 이벤트가 표시되어 볼륨 성능을 모니터링하고 최적화가 필요한 볼륨을 식별할 수 있습니다.

이 기능을 사용하려면 AWS 자격 증명이 필요하며 EDA 대시보드의 지연 시간 메뉴에서 액세스할 수 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html["지연 시간 분석에 대해 자세히 알아보십시오"^].
