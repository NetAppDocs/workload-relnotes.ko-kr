NetApp Workload Factory에 포함된 워크로드의 최신 변경 사항에 대해 알아보세요.



== 2026년 6월 1일



=== 데이터베이스 워크로드



==== Amazon Elastic Block Store의 Oracle 호스트에 대한 맞춤형 절감

NetApp Workload Factory for Databases를 사용하면 Amazon Elastic Block Store(EBS)의 Oracle 호스트에 대한 비용 절감 효과를 사용자 지정할 수 있습니다. 스냅샷 빈도, 복제본 수, 볼륨 세부 정보 등 여러 필드를 조정하여 비용 절감에 미치는 영향을 확인할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html["EBS에서 Oracle 호스트에 대한 절감액 살펴보기"]



==== Microsoft SQL Server 및 PostgreSQL 호스트 배포를 위한 다중 보안 그룹 선택

NetApp Workload Factory for Databases에서는 이제 Microsoft SQL Server 및 PostgreSQL 호스트를 배포할 때 여러 보안 그룹을 선택할 수 있습니다. 이를 통해 데이터베이스 워크로드에 대한 네트워크 보안을 구성할 때 더욱 유연성을 확보할 수 있습니다.

* link:https://docs.netapp.com/us-en/workload-databases/create-database-server.html["Microsoft SQL Server 호스트 배포"]
* link:https://docs.netapp.com/us-en/workload-databases/create-postgresql-server.html["PostgreSQL 호스트 배포"]




==== NetApp Workload Factory에서 AWS GovCloud (US) 지원

NetApp Workload Factory는 AWS GovCloud(미국) 계정을 지원합니다. GovCloud 계정을 사용하면 AWS GovCloud(미국) 리전에 데이터베이스 워크로드를 배포하고 관리할 수 있습니다.



== 2026년 5월 31일



=== NetApp ONTAP용 Amazon FSx



==== NetApp Workload Factory에서 AWS GovCloud (US) 지원

NetApp Workload Factory는 AWS GovCloud(미국) 계정을 지원합니다. GovCloud 계정을 사용하면 AWS GovCloud(미국) 리전에 스토리지 워크로드를 배포하고 관리할 수 있습니다.



==== NetApp Workload Factory에서 볼륨 지연 시간 임계값 설정

볼륨별로 지연 시간 임계값을 구성하여 성능을 모니터링하고 지연 시간 값이 정의된 한계를 초과할 경우 알림을 받을 수 있습니다. Workload Factory는 읽기 및 쓰기 지연 시간을 모두 모니터링하고 두 임계값 중 하나라도 초과되면 알림을 생성합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["NetApp Workload Factory에서 볼륨 지연 시간 임계값 설정"]



==== NetApp Workload Factory에서 복제 지연 추적

NetApp Workload Factory를 사용하면 원격 복제가 구성된 FSx for ONTAP 파일 시스템의 복제 지연을 추적할 수 있습니다. 복제 지연은 데이터가 소스 파일 시스템에 기록된 시점과 대상 파일 시스템으로 복제된 시점 사이의 시간 차이를 나타냅니다. 복제 지연을 추적하면 복제 설정의 상태를 모니터링하고 데이터 일관성 및 복구 시점 목표(RPO)에 영향을 줄 수 있는 잠재적인 문제를 식별하는 데 도움이 됩니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-replication-delays.html["NetApp Workload Factory에서 복제 지연 추적"]



==== NetApp Workload Factory에서 호스트 식별 및 워크로드 정의

Workload Factory는 스토리지를 사용하는 호스트와 해당 호스트 활동과 관련된 워크로드 유형을 식별합니다. 워크로드 정의는 AWS 태그 및 메타데이터를 기반으로 AI에서 생성됩니다. Workload Factory의 파일 시스템, 볼륨 및 블록 장치 리소스 화면에서 호스트 세부 정보와 워크로드 정의를 볼 수 있습니다.



==== NetApp Workload Factory에서 LIF 세부 정보를 확인할 수 있습니다

파일 시스템의 개요 탭에서 모든 파일 시스템 및 스토리지 VM 엔드포인트를 직접 확인할 수 있습니다. Workload Factory 콘솔에서 엔드포인트를 직접 볼 수 있으므로 CLI 또는 CloudShell을 사용하지 않고도 네트워크 액세스, 엔드포인트 및 페일오버 상태를 쉽게 모니터링할 수 있습니다. 역할, 엔드포인트, 노드, IP 주소 및 상태를 포함한 LIF 세부 정보를 표시하려면 파일 시스템에 링크를 연결해야 합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["NetApp Workload Factory에서 파일 시스템에 링크를 연결하는 방법을 알아보세요"].



==== 비용 개선

모든 AWS 계정의 리소스에 대한 비용 분석 결과를 개선하기 위해 새로운 선택적 페이지 수준 필터를 사용할 수 있습니다. 여러 리전을 선택하고 파일 시스템 할당 태그 키 및 키 값을 사용하여 비용 분석 결과를 필터링하고 FSx for ONTAP 파일 시스템 및 볼륨에 대한 리소스별 비용 세부 정보를 확인할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-costs.html["NetApp Workload Factory에서 비용 분석을 위해 필터를 사용하십시오"]



==== NetApp Workload Factory에서 복제 초기화 예약 기능을 사용할 수 있습니다.

데이터 복제 설정 중에 이제 FSx for ONTAP 파일 시스템의 복제 초기화를 예약할 수 있습니다. 복제 관계를 즉시 초기화하거나 지정된 시간에 초기화하는 옵션이 있습니다. 복제 초기화는 소스 파일 시스템의 데이터를 대상 파일 시스템으로 복사하여 복제 기준선을 생성하는 프로세스입니다. 복제 초기화를 예약하면 이 프로세스를 자동화하고 수동 작업 없이 복제 설정을 최신 상태로 유지할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["NetApp Workload Factory에서 데이터를 복제합니다"]



==== FSx for ONTAP 파일 시스템의 Well-Architected 상태에 대한 Excel 보고서

분석이 완료되면 기록 보관용 또는 팀과 공유하기 위해 FSx for ONTAP 파일 시스템의 아키텍처가 잘 구축되었는지에 대한 Excel 보고서를 다운로드할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["NetApp Workload Factory에서 FSx for ONTAP 파일 시스템의 아키텍처 우수성 상태를 확인하세요"]



==== 지연 이벤트 보기 및 분석

Workload Factory에서 이제 FSx for ONTAP 볼륨의 지연 시간 이벤트를 보고 분석할 수 있습니다. 링크 연결 없이도 지연 시간 이벤트를 확인할 수 있어 성능 문제를 더욱 쉽게 모니터링할 수 있습니다. 지연 시간 이벤트를 분석하고 AI 기반 인사이트 및 권장 사항을 받으려면 링크를 사용하여 파일 시스템에 연결해야 합니다. 분석 결과는 현재 지연 시간 값, 임계값, QoS 응답 분석 및 개선 지침을 제공합니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/analyze-events.html["NetApp Workload Factory에서 FSx for ONTAP 이벤트 분석"]



==== 볼륨 지연 시간 임계값 설정

이제 볼륨별로 지연 시간 임계값을 구성하여 성능을 모니터링하고 읽기 또는 쓰기 지연 시간이 설정된 한도를 초과할 경우 알림을 받을 수 있습니다. 지연 시간 모니터링을 통해 워크로드에 영향을 미치기 전에 성능 문제를 사전에 파악할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["NetApp Workload Factory에서 볼륨 지연 시간 임계값 설정"]

link:https://github.com/NetApp/FSx-ONTAP-samples-scripts/tree/main/Management-Utilities/ec2-user-data-iscsi-create-and-mount["블록 디바이스에 태그를 할당하는 EC2 사용자 데이터 스크립트"]

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-capacity-automatically.html["NetApp Workload Factory에서 용량을 자동으로 관리하세요"]

=== 데이터베이스 워크로드



==== Oracle을 위한 잘 설계된 분석

잘 설계된 아키텍처 분석에는 Oracle에 대한 다음과 같은 새로운 구성이 포함됩니다.

* *백업 구성*: Oracle 데이터베이스가 AWS Backup 또는 Amazon FSx for NetApp ONTAP 백업으로 보호되는지 여부를 확인합니다.
* *클론 정리*: 스토리지 비용을 줄이기 위해 오래되었거나 사용하지 않는 클론을 제거하는 것이 좋습니다.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다."]



==== Amazon Elastic Block Store(EBS)의 Oracle 데이터베이스에 대한 자동 절감액 계산

Workload Factory는 Oracle 데이터베이스에 사용되는 EBS 스토리지 비용을 자동으로 분석한 다음 가장 적합하고 비용 효율적인 FSx for ONTAP 구성을 제안합니다. 여러 EBS 호스트를 선택하여 분석한 다음 EBS와 FSx for ONTAP에서 해당 호스트를 실행하는 비용을 비교할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["감지된 EBS 호스트에 대한 비용 절감 효과를 살펴보세요."]



==== CRR 구성 사전 요구 사항 확인

데이터베이스 환경에 대해 지역 간 복제(CRR)를 활성화하면 데이터베이스 워크로드가 FSx for ONTAP 파일 시스템에 연결된 링크 및 복제 설정이 있는지 확인합니다. 필요한 경우 데이터베이스 워크로드는 스토리지와 동일한 방식으로 링크 필수 조건을 처리하도록 FSx for ONTAP 스토리지 워크로드 복제 마법사로 리디렉션합니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다."]



==== Microsoft SQL Server 데이터베이스에 대한 연결된 LUN

이제 Workload Factory의 인벤토리 화면에서 Microsoft SQL Server 데이터베이스에 매핑된 LUN을 볼 수 있습니다. 이를 통해 어떤 LUN이 어떤 데이터베이스와 연결되어 있는지 쉽게 파악하여 스토리지 리소스를 더욱 효율적으로 관리하고 최적화할 수 있습니다.



=== 설정 및 관리



==== Workload Factory의 이메일 알림

이제 Amazon SNS 토픽을 구성하지 않고도 이메일을 통해 Workload Factory 알림을 받을 수 있습니다. 계정에 연결된 모든 이메일 주소로 알림을 보내려면 이메일 알림을 활성화하세요.

Workload Factory 콘솔 또는 NetApp Console에서 이메일 알림을 구성할 수 있습니다. NetApp Console에서는 종 모양 아이콘 아래에 표시되는 NetApp Console 알림을 구성하는 옵션도 제공합니다.

NetApp Console에서 알림을 구성하면 NetApp Console이 알림 전달을 담당하게 됩니다. 중복 알림을 방지하기 위해 Workload Factory 콘솔의 알림 설정은 읽기 전용으로 변경됩니다.



==== EDA 지연 시간에 대한 새 알림

NetApp Workload Factory 알림 서비스에는 EDA 작업의 지연 시간이 특정 임계값을 초과할 때 알려주는 EDA 지연 시간 알림 기능이 포함되어 있습니다.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Workload Factory 알림 구성"]



==== FSx for ONTAP 파일 시스템에 대한 새로운 알림

NetApp Workload Factory 알림 서비스는 계정에 대한 월별 비용 요약 알림을 제공하며, 이를 통해 지출 추세, 비정상적인 변동 사항 및 FSx for ONTAP 파일 시스템에 대한 최적의 비용 절감 기회를 확인할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Workload Factory 알림 구성"]

==== 지연 이벤트에 대한 이메일 및 SNS 알림

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["지연 시간 알림을 구성하는 방법을 알아보세요."^].



==== 대화형 그래프를 사용한 지연 시간 추세 시각화

지연 시간 분석 패널에 이제 포괄적인 지연 시간 분석을 위한 탭 인터페이스가 포함되었습니다.

* *개요*: 지연 시간을 유발하는 구성 요소를 보여주는 자동화된 기본 분석 결과를 확인합니다.
* *시간 경과*: 여러 시간대(1시간, 3시간, 12시간, 24시간 또는 72시간)에 걸쳐 대화형 그래프로 볼륨 지연 시간 동작을 시각화합니다


*시간 경과에 따른 변화* 탭에는 원시 CloudWatch 지연 시간 데이터(알람 발생 원인에 따라 읽기 또는 쓰기)가 경고 및 위험 임계값 선, 위반 표시기, 각 위반에 대한 세부 메트릭과 함께 표시됩니다.

이 포괄적인 시각화를 통해 패턴을 식별하고, 지연 시간 문제가 반복적인지 또는 일회성인지 파악하며, 해결 방법에 대해 더 많은 정보를 바탕으로 의사 결정을 내릴 수 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["지연 시간 추세를 분석하는 방법을 알아보십시오"^].

==== Oracle을 위한 잘 설계된 분석

잘 설계된 아키텍처 분석에는 Oracle에 대한 다음과 같은 새로운 구성이 포함됩니다.

* *지역 간 복제*: Oracle 데이터베이스를 제공하는 FSx for ONTAP 파일 시스템에 대해 지역 간 복제(CRR)가 활성화되어 있는지 확인합니다.
* *애플리케이션 정합성 보장 스냅샷*: Oracle 데이터베이스가 NetApp SnapCenter를 통해 애플리케이션 정합성 보장 스냅샷으로 보호되는지 확인합니다.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다."]



==== Oracle에 대한 일회성 평가

Workload Factory는 이제 필요할 때마다 Oracle 데이터베이스 배포 구성을 분석할 수 있는 일회성 평가 기능을 제공합니다(신규 배포, 마이그레이션 후, 문제 해결 등). 배포 구성을 최적화하고 모범 사례를 준수하는 데 도움이 되는 권장 사항을 받아볼 수 있습니다. 이 기능을 통해 설정 프로세스를 완전히 수행하지 않고도 잘 설계된 기능을 사용해 볼 수 있습니다. Workload Factory에 자격 증명을 저장하거나, 데이터베이스 리소스를 등록하거나, 외부 연결을 설정하지 않고도 데이터베이스를 평가할 수 있습니다.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Oracle 데이터베이스 배포에 대한 일회성 평가를 수행하십시오"]

==== EDA 용량 권장 사항

Workload Factory는 이제 EDA 인벤토리에 있는 각 FSx for ONTAP 클러스터의 SSD 용량을 자동으로 분석하고 조정이 필요할 때 용량 권장 사항을 생성합니다. 알고리즘은 각 클러스터에 대해 24시간마다 한 번씩 실행됩니다.

권장 사항이 확인되면:

* 즉시 이메일 또는 WAD 알림을 받게 됩니다(주간 요약에는 포함되지 않음).
* Clusters 테이블에서 영향을 받는 클러스터에 전구 표시등이 나타납니다.
* Overview 대시보드의 요약에는 활성 권장 사항이 있는 클러스터의 총 수가 표시됩니다.


활성 권장 사항이 있는 클러스터를 선택하여 권장 사항을 확인하세요. 권장 사항에는 제안된 변경 사항과 그 이유가 설명되어 있습니다.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["용량 권장 사항에 대해 자세히 알아보십시오"^].

=== VMware 워크로드



==== Google Cloud VMware Engine용 마이그레이션 어드바이저 지원 소개

NetApp Workload Factory가 이제 Google Cloud VMware Engine을 지원합니다. Google Cloud NetApp Volumes를 외부 NFS 데이터스토어로 사용하여 온프레미스 VMware 워크로드를 Google Cloud VMware Engine으로 마이그레이션할 수 있습니다. 이를 통해 애플리케이션을 재구성하거나 플랫폼을 변경할 필요 없이 비용을 최적화하고 VMware 환경을 더욱 효과적으로 제어할 수 있습니다.

Google Cloud VMware Engine 마이그레이션 어드바이저는 다음을 제공합니다.

* VE1 및 VE2 노드 유형을 지원하는 자동화된 Google Cloud VMware Engine 클러스터 크기 조정 및 설계
* vSAN 내부 데이터스토어와 Google Cloud NetApp Volumes 외부 NFS 데이터스토어 간의 VM 레이아웃 최적화.
* vSAN 단독 구성과 Google Cloud NetApp Volumes 강화 구성 간의 상세한 비용 비교 및 총소유비용(TCO) 분석.
* Standard, Premium 및 Extreme Google Cloud NetApp Volumes 서비스 레벨을 지원합니다.
* Terraform을 사용한 자동화된 프로비저닝을 위한 Infrastructure-as-Code 배포.
